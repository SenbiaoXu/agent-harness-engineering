# 报告模板参考

本文件包含各阶段使用的输出模板。在对应阶段需要生成报告时，按以下格式输出。

## 项目分析报告

```
## 项目概况
- 项目名称/用途
- 源语言及版本要求
- 项目规模（文件数、代码行数的大致量级）

## 项目结构
（目录树 + 各目录/文件的职责说明）

## 依赖清单
（每个外部依赖的名称、版本、用途）

## 功能模块清单
### 模块1：[模块名]
- 职责：...
- 对外接口：...
- 核心实现逻辑：...
- 依赖的其他模块：...

### 模块2：[模块名]
...

## 入口与启动流程
（从入口文件到各模块的调用链）

## 构建与运行
- 构建命令：...
- 运行命令：...
- 环境要求：...
```

## 功能差异报告

当发现原项目与新项目之间存在不一致时，按此格式逐模块报告：

```
> ## 功能差异报告
> ### [模块名]
> - **缺失功能**：[具体描述]
> - **行为差异**：[具体描述]
> - **建议**：[修复方案]
```

## 转换完成报告

```
## 转换完成报告

### 项目信息
- 源语言 → 目标语言
- 原项目路径 → 新项目路径

### 模块对照表
| 原模块 | 新模块 | 状态 |
|--------|--------|------|
| ... | ... | 已对应 |

### 构建结果
- 构建命令：...
- 构建状态：成功/失败

### 运行结果
- 运行命令：...
- 运行状态：成功/失败
- 测试结果：x/y 通过

### 已知差异（用户确认的）
- ...
```

## 转换进度跟踪（CONVERSION_STATUS.md）

大型或跨会话的转换项目，在目标项目根目录维护此文件，用于记录进度和关键决策。

```
# 转换进度

## 项目信息
- 源语言 → 目标语言
- 原项目路径 → 新项目路径
- 开始日期：YYYY-MM-DD

## 模块进度

| 模块名 | 源文件 | 目标文件 | 状态 | 方案选择 | 备注 |
|--------|--------|----------|------|----------|------|
| 模块A | src/a.py | lib/a.go | 已完成 | 方案1（标准库实现） | |
| 模块B | src/b.py | lib/b.go | 进行中 | 方案2（第三方库） | 需处理异步逻辑 |
| 模块C | src/c.py | — | 待转换 | 待确认 | 依赖模块B |

## 配置与基础设施

| 项目 | 状态 | 备注 |
|------|------|------|
| 项目骨架 | 已完成 | go mod init |
| 构建配置 | 待转换 | |
| CI 配置 | 待转换 | |
| 环境配置 | 待转换 | |

## 关键决策记录
- [YYYY-MM-DD] 模块A：选择方案1而非方案2，原因是...
- [YYYY-MM-DD] 依赖库选择：使用 X 替代 Y，差异是...

## 待解决问题
- [ ] 问题描述...
```
